在游戏的发展历史中，出现过哪些有意思的 加密反盗版机制和破解机制？
高大勇
任天堂曾经出过 Famicon Disk System 磁碟系统，它使用了磁盘，游戏磁盘本身上面有一行大大的突起的 NINTENDO 字样，其实它是有实际作用的——这就是反盗版措施！Disk System 里面有一个卡槽，正好能跟磁盘上的这个 Logo 吻合上，只有吻合了，游戏才能开始玩。
图片来源
于是玩家们除了直接购买正版游戏之外，还可以事先购买任天堂发售的磁盘，到遍布日本的 3000 台磁盘拷贝机上去购买并拷贝游戏。
可是人们很快就发现，通过个人电脑可以轻易的对磁盘进行复制，操作步骤很简单，以至于后来连各种杂志都在教大家复制磁盘的方法……


接下来我来回忆一下早期遇到过的关于磁盘系统的反盗版，可能不是那么精确，但是意思不会错。
Apple ][ 最流行的时候，软盘也随之成为最流行的数据载体，而那个时候除了我们自用普通的数据盘可以随意存取文件外，很多我们拿到手的正版软件／游戏盘都是做了防盗版处理的。那个时候，一个软件／游戏就是一张或者多张磁盘，我们将磁盘放入驱动器后，要么重新 RESET 系统，直接从磁盘启动，要么就是敲入 PR#6 启动磁盘。
在启动过程中，磁盘控制器 ROM 中会有一段程序执行，将磁头移动到 Track 0，寻找固定的数据 D5 AA 96，找到之后就会读取 256 字节的数据，而这部分数据被认为是可执行的，进而执行它，这个程序就是磁盘的 bootstrap 程序，后面的控制权则由它来决定。
按照正常的情况，bootstrap 会直接运行磁盘上的对应程序，软件或者游戏就跑起来了。
由于这部分程序是可以自行编写的，于是，各种各样的花样就出现了，甚至可以说是脑洞大开，比如：
修改扇区的起始和结束标记。前面说到 D5 AA 96，这是标准的扇区起始标记，一般在复制文件的时候都是先找到起始位置，读取到结束位置，然后再读取下一个扇区。可是如果开发者修改了这个标记，比如变成 DD DD DD，那么自己编写的 bootstrap 能够识别它，但是标准的复制程序就识别不了了。于是就会造成文件就是无法复制出来；
部分兼容标准系统，部分不兼容。比如，能列目录，但是列目录列出来的居然是使用说明，完全看不到文件列表；
修改扇区的 checksum 校验算法。系统对扇区数据的校验是有自己的一套简单算法的，但是开发者们可以在 bootstrap 中修改它，自己用一套算法。这样在正常系统里面复制磁盘上的文件的时候，看上去文件都没问题，但是复制的时候会不断出现校验错误，无法完成；
利用磁盘上未用到的地方。一张磁盘是按照行业标准来格式化的，其中有一些位置是不被用到的，如果开发者想要读取这些部分来做检查，也是可行的。这样不会影响正常的文件复制，但是复制完了会发现就是运行不了，因为正版程序里面检查的地方，是正常系统根本不会去存取的地方。
以上只是一些方法，其实当时用的方法五花八门，破解者必须要先读懂磁盘的 bootstrap 代码，才能想出办法来对应。不过呢，最后也全都被破解了。
随着时间推移，出现了类似 Locksmith、Copy ][ 这样的整盘步进复制软件，前面说到的很多防盗版方式就不存在了。
众多的拷贝工具
图片来源
可是这个斗争远没有停止，反盗版者还是在不断的想出新办法来，比如：
为正版磁盘标记坏道。在磁盘上标记一些坏道，正版的程序知道哪里有坏道，可以避开，而全盘复制的时候，遇到坏道会产生错误，以至于无法复制整盘。这个方法在 DVD 中也有用到；
那么后来还是有方法能够忽略坏道复制，所以，开发者又想出了新的办法，既然知道哪里标记了坏道，那么可以刻意的去读取，如果失败了就对了。而跳过保护正确复制了整盘的软件在执行到这里的时候反而会很顺利的不出错，那么不出错的就是盗版。这种要破解的话就要改代码了，不过在那个时候来说也不太难；
在磁盘上定义重复编号的扇区。正常情况下扇区是有序的，不会有重复现象，但是刻意标记重复扇区编号，并且去读取这个扇区数据的话，由于磁盘一直在旋转，所以两次读取同一编号的扇区返回结果应该是不同的，而显然自以为正确复制出来的盗版软件在读取这部分内容的时候永远是一致的，因此就被判定为盗版了。
在磁盘上能做的手脚远不止这些，我还听说过有人通过物理破坏磁盘扇区的方式来做加密，不过没见到过。以上提到的这些方式我小的时候多多少少见过一些，那个时代真是百花齐放的黄金时代。

Frontwing
bilibili
Crunchyroll
TOP MARSHAL
MAGES.
Light future
STUDIO MAUSU

本作委员会主要是由游戏原作方（Frontwing）、网络播放方（bilibili、Crunchyroll），碟片、音乐发行公司（TOP MARSHAL、MAGES.）主要牵头，电视台参与比重很小，因此并没有收到电视台约束。
而如上图所示bilibili是制作委员会的第二位，按照制委默认的规矩，bilibili是投资占比中排第二的大头有着极高的话语权。这个季度bilibili有三部投资的动画，分辨是Island、百链霸王与圣约女武神、春原庄的管理人小姐，而这三部在国内都由bilibili负责代理。


那么回到问题，为什么bilibili不独播Island呢？
因为bilibili在制作委员会里有着非常高的地位，因此Island动画带来的收益bilibili是能拿到相当多的分成的。而bilibili把握着国内的动画代理权，卖给其他网站动画播放权所带来收益可以让bilibili获得可观的额外收入，回收投资成本。

董玉龙
首先，为了让刻画系统性质的变量尽可能简单，我们选取广义坐标、动量（相空间 坐标），用哈密顿力学描述动力学性质。其实就是把牛顿力学的约束全部去掉了
对于我们希望观察到的力学量性质，可以表示成相空间坐标的函数，这个函数并不显含时间，对时间的依赖体现在相空间坐标对时间的依赖。Liouville算子及时间演化方程为
形式上可以将力学量的时间演化方程的解写成 ，其中， 是相流。Liouville算子可以诱导一个单参微分同胚群，群元可以做为相空间上的测度的推前映射、力学量拉回映射。由对偶性，可以得到相空间上的测度的时间演化
Note.
(1) 如果学过随机过程，可以发现，这里的Liouville算子和随机过程的生成元很类似，诱导了转移半群
(2) 这里的相流是确定的，也就是如果给定了哈密顿量，给定了初始条件，系统的演化是完全确定的
下面，我们开始考虑为什么会出现随机性
对于实际的系统，自由度都是非常大的，我们不可能求解那样高维的哈密顿方程去得到系统的相流，所以，即使我们已经通过对偶性得到了测度的表达式，因为没有相流，也没办法应用。但是，实际的问题中，我们往往只对系统的一部分自由度感兴趣，因此，我们现在需要考虑的问题是，如何从十分复杂的系统中得到一个比较小的子系统的运动规律并且尽可能在保持整个系统性质的前提下去简化计算量
举个例子，对于水里的一个小花粉粒，我们只想知道花粉粒的运动，对整杯水的运动并不感兴趣

如何理解自由能？
对于非平衡系综，因为给定样本点之后，随机过程是时间的函数，给出了一条样本轨道，所以上面的熵、自由能等全部改成样本轨道的泛函就可以了。那么，下面我们要说的就是怎么从微观过渡到宏观，即热力学性质。热力学性质实际上就是系统的最概然性质，前面提到了相空间坐标可以作为微观态，给定NVE下熵的最大值（规范化之后为0）的点的集合为宏观态，给定NVT下自由能的最小值的点的集合为宏观态。在热力学极限下，力学量的取值会依概率的收敛到宏观态。所以，要将统计物理的结论过渡到热力学，就是把有限自由度系统的分布性质先研究清楚，给出宏观态集合，这些值就是热力学极限下系统的热力学性质的体现

在游戏的发展历史中，出现过哪些有意思的 加密反盗版机制和破解机制？
高大勇
任天堂曾经出过 Famicon Disk System 磁碟系统，它使用了磁盘，游戏磁盘本身上面有一行大大的突起的 NINTENDO 字样，其实它是有实际作用的——这就是反盗版措施！Disk System 里面有一个卡槽，正好能跟磁盘上的这个 Logo 吻合上，只有吻合了，游戏才能开始玩。
图片来源
于是玩家们除了直接购买正版游戏之外，还可以事先购买任天堂发售的磁盘，到遍布日本的 3000 台磁盘拷贝机上去购买并拷贝游戏。
可是人们很快就发现，通过个人电脑可以轻易的对磁盘进行复制，操作步骤很简单，以至于后来连各种杂志都在教大家复制磁盘的方法……


接下来我来回忆一下早期遇到过的关于磁盘系统的反盗版，可能不是那么精确，但是意思不会错。
Apple ][ 最流行的时候，软盘也随之成为最流行的数据载体，而那个时候除了我们自用普通的数据盘可以随意存取文件外，很多我们拿到手的正版软件／游戏盘都是做了防盗版处理的。那个时候，一个软件／游戏就是一张或者多张磁盘，我们将磁盘放入驱动器后，要么重新 RESET 系统，直接从磁盘启动，要么就是敲入 PR#6 启动磁盘。
在启动过程中，磁盘控制器 ROM 中会有一段程序执行，将磁头移动到 Track 0，寻找固定的数据 D5 AA 96，找到之后就会读取 256 字节的数据，而这部分数据被认为是可执行的，进而执行它，这个程序就是磁盘的 bootstrap 程序，后面的控制权则由它来决定。
按照正常的情况，bootstrap 会直接运行磁盘上的对应程序，软件或者游戏就跑起来了。
由于这部分程序是可以自行编写的，于是，各种各样的花样就出现了，甚至可以说是脑洞大开，比如：
修改扇区的起始和结束标记。前面说到 D5 AA 96，这是标准的扇区起始标记，一般在复制文件的时候都是先找到起始位置，读取到结束位置，然后再读取下一个扇区。可是如果开发者修改了这个标记，比如变成 DD DD DD，那么自己编写的 bootstrap 能够识别它，但是标准的复制程序就识别不了了。于是就会造成文件就是无法复制出来；
部分兼容标准系统，部分不兼容。比如，能列目录，但是列目录列出来的居然是使用说明，完全看不到文件列表；
修改扇区的 checksum 校验算法。系统对扇区数据的校验是有自己的一套简单算法的，但是开发者们可以在 bootstrap 中修改它，自己用一套算法。这样在正常系统里面复制磁盘上的文件的时候，看上去文件都没问题，但是复制的时候会不断出现校验错误，无法完成；
利用磁盘上未用到的地方。一张磁盘是按照行业标准来格式化的，其中有一些位置是不被用到的，如果开发者想要读取这些部分来做检查，也是可行的。这样不会影响正常的文件复制，但是复制完了会发现就是运行不了，因为正版程序里面检查的地方，是正常系统根本不会去存取的地方。
以上只是一些方法，其实当时用的方法五花八门，破解者必须要先读懂磁盘的 bootstrap 代码，才能想出办法来对应。不过呢，最后也全都被破解了。
随着时间推移，出现了类似 Locksmith、Copy ][ 这样的整盘步进复制软件，前面说到的很多防盗版方式就不存在了。
众多的拷贝工具
图片来源
可是这个斗争远没有停止，反盗版者还是在不断的想出新办法来，比如：
为正版磁盘标记坏道。在磁盘上标记一些坏道，正版的程序知道哪里有坏道，可以避开，而全盘复制的时候，遇到坏道会产生错误，以至于无法复制整盘。这个方法在 DVD 中也有用到；
那么后来还是有方法能够忽略坏道复制，所以，开发者又想出了新的办法，既然知道哪里标记了坏道，那么可以刻意的去读取，如果失败了就对了。而跳过保护正确复制了整盘的软件在执行到这里的时候反而会很顺利的不出错，那么不出错的就是盗版。这种要破解的话就要改代码了，不过在那个时候来说也不太难；
在磁盘上定义重复编号的扇区。正常情况下扇区是有序的，不会有重复现象，但是刻意标记重复扇区编号，并且去读取这个扇区数据的话，由于磁盘一直在旋转，所以两次读取同一编号的扇区返回结果应该是不同的，而显然自以为正确复制出来的盗版软件在读取这部分内容的时候永远是一致的，因此就被判定为盗版了。
在磁盘上能做的手脚远不止这些，我还听说过有人通过物理破坏磁盘扇区的方式来做加密，不过没见到过。以上提到的这些方式我小的时候多多少少见过一些，那个时代真是百花齐放的黄金时代。

你要知道，地球表面的这些皱纹，才是地球年轻富有活力的标志，地球内部的力量导致地表隆升，而地球表面的各种流水、风、雨又雕刻出了这些皱纹，当你看到这些皱纹的时候你应该开心才对。我们的地球内部还有丰富的地热，板块不断移动，碰撞或者增生，火山喷发，已经形成岩石的物质还能回到地表的循环中，不像火星一样已经冷却下来，只是一片死寂。地表的水和大气在太阳光与地球重力的共同作用下也发生着循环。丰富的水，氧气和各种矿物滋养着地球上的生命，我感觉地球现在真是好的不能再好了。
可能答主是说这些山像皱纹一样，是指植被覆盖少吧，可你要知道，在二叠纪晚期和三叠纪盘古大陆形成的时候，大陆内部是一篇无边无际的大沙漠，干燥而炽热，比现在地球上的条件恶劣多了，大部分生物只能生活在沿海地区，以人类的眼光看来那时的条件比现在差的多，那么热那么干没有植被怎么住人啊？但是各种生物照样在地球上生活的有滋有味。
那究竟什么时候我感觉地球很脆弱呢？当我一想到地球的尺度和宇宙比起来是无尽的渺小的时候，我觉得地球很脆弱，这么说吧，地球自身以及地球表面的各种活动都不足以毁灭地球自己，只有来自宇宙的力量才能毁灭地球，比如再来一次火星大小的行星撞击地球，就像当初月球形成时那样，或者伽马射线暴的方向正好指向地球，或者随着太阳年龄增大，自身膨胀，导致地球越来越热，水全部蒸发掉。各种你能想到的宇宙尺度的灾难事件都有可能导致地球上整个生态系统的毁灭，这些宇宙事件也不是人类有能力可以阻止的，现在考虑它们也没用。当然地球上的生命对地球来说没有任何意义，地球本身没有情感，没有说像母亲一样要呵护这些生命，一切的形成都是自然规律的结果，地球的存在对于宇宙也没有什么影响，只是我们这些地球上的小生物很在意罢了。

当然这个答案的意思不是说保护环境不重要，我们当然要保护环境，不过是为了我们人类能生存下去，只有人类生存下去我们才有机会进一步探索各种科学问题，发展各种科技，从而在以后科技水平能够达到真正的保护地球，将一切有威胁的天体碎成渣渣，逆转恒星的衰老和死亡，甚至把地球搬到其他的地方，所以说保护环境就是保护地球也没错，这就是“曲线救球” 。

如何理解自由能？
对于非平衡系综，因为给定样本点之后，随机过程是时间的函数，给出了一条样本轨道，所以上面的熵、自由能等全部改成样本轨道的泛函就可以了。那么，下面我们要说的就是怎么从微观过渡到宏观，即热力学性质。热力学性质实际上就是系统的最概然性质，前面提到了相空间坐标可以作为微观态，给定NVE下熵的最大值（规范化之后为0）的点的集合为宏观态，给定NVT下自由能的最小值的点的集合为宏观态。在热力学极限下，力学量的取值会依概率的收敛到宏观态。所以，要将统计物理的结论过渡到热力学，就是把有限自由度系统的分布性质先研究清楚，给出宏观态集合，这些值就是热力学极限下系统的热力学性质的体现
上面写了这么多，实际上就只是一个思路而已，要直接通过哈密顿力学计算热力学是非常困难的，因为，里面涉及到很多的假设，并且计算量也是非常的大。热力学、统计物理之所以是一个独立的分支，而不是理论力学的从属，也是因为各自的研究方法各有不同，并不是单纯的属于关系。不过，在研究里还是比较有用的，比如Monte Carlo就是用哈密顿力学／子系统的哈密顿力学（Langevin方程）来得到相空间坐标的样本轨道，然后研究力学量的性质，或者和实验得到的宏观性质对比
